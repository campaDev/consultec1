# Requerimientos del Cliente: Proyecto de Renovación Consultec

Para cumplir con el cronograma de **8 semanas** y alcanzar los estándares de **Lighthouse 100%**, es fundamental la provisión de los siguientes recursos.

## 1. Identidad de Marca (Semana 2)
Para la fase de diseño UI/UX, se requiere:
* **Logotipos:** Archivos en alta resolución (SVG o AI preferiblemente).
* **Manual de Marca:** Definición de colores y tipografías (si existen).
* **Fotografías:** Imágenes reales del equipo y local para generar confianza regional (Social Proof).

## 2. Accesos Técnicos (Semana 1 y 8)
* **Dominio:** Acceso al panel de control (ej. Nic.py o similar) para la configuración de DNS al finalizar el proyecto.
* **WordPress Actual:** Credenciales temporales para la extracción y migración de contenidos existentes.
* **Herramientas de Google:** Acceso a Search Console y Analytics para asegurar la continuidad del SEO y GEO-posicionamiento.

## 3. Información del Catálogo (Límite: Semana 3)
La información debe estructurarse según el modelo de datos definido para el motor de Astro:

### 3.1 Software (Sistemas Aguarai)
* **Datos Clave:** Nombre del sistema, descripción de módulos y precio en Guaraníes.
* **Segmentación:** Definición de industria (Retail, Gastronomía, Industria, etc.).

### 3.2 Hardware (Equipamiento)
* **Ficha Técnica:** Marca, modelo y especificaciones de rendimiento.
* **Multimedia:** Fotografías con fondo limpio para optimización AVIF/WebP.

## 4. Matriz de Entregas Críticas

| Plazo Máximo | Requerimiento | Impacto en el Proyecto |
| :--- | :--- | :--- |
| **Fin Semana 1** | Accesos y Logos | Inicio de arquitectura y diseño UI. |
| **Fin Semana 3** | Datos de Catálogo | Construcción del motor de productos. |
| **Fin Semana 5** | Casos de Éxito | Optimización de IA (AIO) y contenidos SEO. |

## 5. Requerimientos Estratégicos (Complementarios)

Para potenciar la autoridad y el cierre de ventas, se solicita:

* **Partnerships:** Logos y breves descripciones de las alianzas actuales (Gadget, Aguará-soft, etc.).
* **Social Proof (Prueba Social):** Listado de logos de clientes actuales y, opcionalmente, 2-3 testimonios breves.
* **Recursos Descargables:** Si existen manuales de usuario en PDF o folletos digitales de los sistemas, para incluirlos como material de valor.
* **Datos de Conversión:** Definición de a qué correos deben llegar los formularios y qué números de WhatsApp se asignarán a "Ventas" y "Soporte Técnico".
* **Scripts de Terceros:** Si la empresa realiza publicidad, se requerirán los ID de Meta Pixel o Google Tags para integrarlos de forma nativa en Astro.

---
**Nota:** El cumplimiento de estas fechas es esencial para garantizar que la web sea estable, rápida y esté lista para el lanzamiento oficial en la **Semana 8**.
